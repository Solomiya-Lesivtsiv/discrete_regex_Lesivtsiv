# Лабораторна робота: Регулярні вирази та Скінченні автомати

## Опис завдання

Реалізація детермінованого скінченного автомата (DFA) для перевірки рядків за регулярними виразами. Система підтримує базові оператори регулярних виразів:
- `.` - будь-який символ
- `*` - нуль або більше входжень
- `+` - один або більше входжень
- ASCII символи та цифри

State (ABC) ├── StartState - початковий стан ├── TerminationState - кінцевий стан ├── AsciiState - стан для конкретного символу ├── DotState - стан для будь-якого символу ├── StarState - модифікатор * (0 або більше) └── PlusState - модифікатор + (1 або більше)

### Основний клас `RegexFSM`

```python
RegexFSM
├── __init__(regex_expr)        - побудова автомата з регулярного виразу
├── __init_next_state()         - створення стану для кожного символу
└── check_string(text)          - перевірка рядка проти регулярного виразу
    └── get_epsilon_closure()   - розширення станів через епсилон-переходи
```

#1. Побудова автомата
Алгоритм читає регулярний вираз символ за символом:

Звичайні символи: створюють AsciiState або DotState та додаються до ланцюга
Модифікатор *: створює StarState, який дозволяє нуль або більше входжень
Модифікатор +: створює PlusState, який вимагає щонайменше один вхід

#2. Перевірка рядка
Використовується NFA з епсилон-замиканням (epsilon closure):

Розширити початкові стани через епсилон-переходи
Для кожного символу рядка:
Знайти всі стани, в які можна перейти зі поточних станів
Розширити нові стани через епсилон-переходи
Перевірити, чи досягли TerminationState

#Приклад використання
```python
regex_pattern = "a*4.+hi"

regex_compiled = RegexFSM(regex_pattern)

print(regex_compiled.check_string("aaaaaa4uhi"))  # True
print(regex_compiled.check_string("4uhi"))  # True
print(regex_compiled.check_string("meow"))  # False
```
